# Contribuindo

Este projeto segue uma abordagem de desenvolvimento por sprints, com foco em evolução incremental e documentação clara.

## Fluxo recomendado
1. Crie uma branch para a sprint ou funcionalidade.
2. Implemente mudanças pequenas e bem testadas.
3. Atualize o changelog e o roadmap quando houver evolução relevante.
4. Mantenha commits pequenos e com mensagens claras.

## Padrões de commit
- `feat:` para novas funcionalidades
- `fix:` para correções
- `docs:` para documentação
- `refactor:` para reorganização de código
- `test:` para testes
